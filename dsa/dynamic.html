DSA revision guide

Dynamic Programming

DP is just recursion with memory. Store results you have already computed so you never solve the same subproblem twice. Once you have a recursive solution, adding memoisation is almost mechanical.

01 — What is Dynamic Programming?

Simple definition: DP = recursion + caching. If a recursive solution solves the same subproblem more than once, store the result the first time and reuse it. That is the whole idea. The fancy name hides something simple.

Overlapping subproblems
Same subproblem solved multiple times
fib(5) needs fib(3). fib(4) also needs fib(3). Without caching, fib(3) computed twice — and fib(2) many more times. DP stores it once.
Optimal substructure
Best answer uses best sub-answers
The optimal solution to the whole problem contains optimal solutions to subproblems. Shortest path A to C via B needs shortest A-B and shortest B-C.
Memoisation vs Tabulation
Top-down vs bottom-up
Memoisation = recursion + cache (top-down). Tabulation = loops + table (bottom-up). Same answer, different direction.
DP vs Greedy
When do you need DP?
Greedy picks one best option and never looks back. DP considers ALL options and picks the best. If greedy fails — use DP.
Why caching works
Fibonacci: O(2^n) vs O(n)
Without cache: billions of redundant calls for n=50. With cache: n distinct subproblems, each solved once. Click to see the call tree.
02 — The 4-step framework
Every DP problem follows the same 4 steps. Master this process and DP becomes pattern recognition, not magic.
Universal DP process
1
Define the state. What does dp[i] mean? Usually: "best answer considering the first i elements." For 2D: "answer using s1[:i] and s2[:j]." This is the hardest step.
2
Write the recurrence. How does dp[i] relate to smaller subproblems? Ask: what choices do I have at this step? Try each. Take the best (min/max) or sum them.
3
Identify base cases. What is the answer for the smallest input? dp[0] = 0? dp[1] = nums[0]? Wrong base cases produce wrong answers silently.
4
Determine the answer. Which dp value is the final answer? Usually dp[n]. Sometimes max(dp). Sometimes dp[m][n].
Top-down (memoisation)
Start with recursion. Add @cache. Done.
from functools import cache

def fib(n):
    @cache
    def dp(i):
        if i <= 1: return i    # base case
        return dp(i-1) + dp(i-2)  # recurrence
    return dp(n)            # answer
Bottom-up (tabulation)
Fill a table from smallest to largest.
def fib(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[0], dp[1] = 0, 1  # base cases
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]  # recurrence
    return dp[n]           # answer
03 — 1D DP patterns
The simplest DP problems. dp is a 1D array. Each cell depends on a few previous cells. Master these before moving to 2D.
Fibonacci pattern
dp[i] depends on dp[i-1] and dp[i-2]
# LC 70 — Climbing Stairs
# dp[i] = ways to reach step i
# dp[i] = dp[i-1] + dp[i-2]
dp = [0] * (n + 1)
dp[1] = 1; dp[2] = 2
for i in range(3, n + 1):
    dp[i] = dp[i-1] + dp[i-2]
return dp[n]
Include / Exclude pattern
Take it or leave it — no adjacent
# LC 198 — House Robber
# dp[i] = max money from first i houses
# rob i:  nums[i] + dp[i-2]
# skip i: dp[i-1]
prev2, prev1 = 0, 0
for num in nums:
    curr = max(prev1, prev2 + num)
    prev2, prev1 = prev1, curr
return prev1
Kadane — Max Subarray
LC 53 — extend current or start fresh
# dp[i] = max sum subarray ENDING at i
# either extend previous or restart at nums[i]
curr = best = nums[0]
for num in nums[1:]:
    curr = max(num, curr + num)
    best = max(best, curr)
return best
Coin Change — LC 322
Min coins to make amount — try each coin
# dp[a] = min coins for amount a
dp = [float("inf")] * (amount + 1)
dp[0] = 0  # base: 0 coins to make 0
for a in range(1, amount + 1):
    for coin in coins:
        if coin <= a:
            dp[a] = min(dp[a], dp[a-coin] + 1)
return dp[amount] if dp[amount] != float("inf") else -1
Longest Increasing Subsequence — LC 300
dp[i] = length of LIS ending at index i. For each i, look back at every j < i where nums[j] < nums[i].
# dp[i] = LIS length ending at nums[i]
dp = [1] * len(nums)  # every element alone = length 1
for i in range(1, len(nums)):
    for j in range(i):
        if nums[j] < nums[i]:
            dp[i] = max(dp[i], dp[j] + 1)
return max(dp)  # answer is the global max
04 — 2D DP patterns
Two-dimensional DP. dp[i][j] represents the answer involving two inputs — two strings, a grid position, or items vs capacity.
Unique Paths — LC 62
dp[i][j] = ways to reach (i,j) moving only right or down.
# dp[i][j] = dp[i-1][j] + dp[i][j-1]
m, n = len(grid), len(grid[0])
dp = [[1] * n for _ in range(m)]
for i in range(1, m):
    for j in range(1, n):
        dp[i][j] = dp[i-1][j] + dp[i][j-1]
return dp[m-1][n-1]
Min Path Sum — LC 64
dp[i][j] = min cost to reach (i,j). Comes from above or left.
# init first row and column as base cases
for j in range(1,n): dp[0][j]=dp[0][j-1]+grid[0][j]
for i in range(1,m): dp[i][0]=dp[i-1][0]+grid[i][0]
for i in range(1, m):
    for j in range(1, n):
        dp[i][j] = grid[i][j] + min(dp[i-1][j],
                                     dp[i][j-1])
return dp[m-1][n-1]
Interval DP
dp[i][j] = answer for range i..j
Fill by increasing length. Try every split point k. Burst Balloons, Palindromic Subsequence, Matrix Chain.
What dp[i][j] means
Common 2D state interpretations
String: first i chars of s1, first j of s2. Knapsack: first i items, capacity j. Grid: row i, col j.
Grid DP tips
Obstacles, reverse fill, base cases
Obstacles: set dp=0 at blocked cell. Reverse fill: Dungeon Game fills bottom-right to top-left. First row and col are always base cases.
05 — Knapsack patterns
The most important DP family. 0/1 knapsack, unbounded knapsack, and subset sum all share the same skeleton.

Core idea: For each item decide — include it or exclude it. If included, remaining capacity = W - weight[i]. dp[i][w] = best value using first i items with capacity w.

0/1 Knapsack — each item at most once 
# dp[i][w] = max value, first i items, cap w
# exclude item i: dp[i-1][w]
# include item i: val[i] + dp[i-1][w - wt[i]]
dp = [[0]*(W+1) for _ in range(n+1)]
for i in range(1, n+1):
    for w in range(W+1):
        dp[i][w] = dp[i-1][w]           # exclude
        if wt[i-1] <= w:               # can include?
            dp[i][w] = max(dp[i][w],
                val[i-1] + dp[i-1][w-wt[i-1]])
return dp[n][W]
1D optimised — iterate BACKWARDS 
# Collapse 2D to 1D — same result
# KEY: go right-to-left for 0/1 knapsack
# (prevents using same item twice)
dp = [0] * (W + 1)
for i in range(n):
    for w in range(W, wt[i]-1, -1):  # <-- backwards
        dp[w] = max(dp[w], val[i] + dp[w-wt[i]])
return dp[W]

# Unbounded (infinite items): go LEFT to RIGHT
# for w in range(wt[i], W+1): same formula
Subset Sum — LC 416
Can we pick elements that sum to target?
# LC 416 — Equal Partition Subset Sum
target = sum(nums) // 2
dp = [False] * (target + 1)
dp[0] = True  # empty set sums to 0
for num in nums:
    for t in range(target, num-1, -1):
        dp[t] = dp[t] or dp[t - num]
return dp[target]
Coin Change II — LC 518
Count combinations (loop order matters!)
# coins outer, amounts inner = combinations
# (amounts outer = permutations — different!)
dp = [0] * (amount + 1)
dp[0] = 1  # 1 way to make 0: do nothing
for coin in coins:
    for a in range(coin, amount + 1):
        dp[a] += dp[a - coin]
return dp[amount]
06 — String DP
String DP almost always uses a 2D table. dp[i][j] involves s1[:i] and s2[:j]. The recurrence splits on whether characters match.
LCS — Longest Common Subsequence 
# dp[i][j] = LCS of s1[:i] and s2[:j]
# if chars match: dp[i-1][j-1] + 1
# else: max(skip s1 char, skip s2 char)
m, n = len(s1), len(s2)
dp = [[0]*(n+1) for _ in range(m+1)]
for i in range(1, m+1):
    for j in range(1, n+1):
        if s1[i-1] == s2[j-1]:
            dp[i][j] = dp[i-1][j-1] + 1
        else:
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])
return dp[m][n]
Edit Distance — LC 72 
# dp[i][j] = min edits: s1[:i] into s2[:j]
# match: dp[i-1][j-1] (free)
# insert/delete/replace: 1 + min of neighbors
m, n = len(s1), len(s2)
dp = [[0]*(n+1) for _ in range(m+1)]
for i in range(m+1): dp[i][0] = i  # delete all
for j in range(n+1): dp[0][j] = j  # insert all
for i in range(1, m+1):
    for j in range(1, n+1):
        if s1[i-1] == s2[j-1]:
            dp[i][j] = dp[i-1][j-1]
        else:
            dp[i][j] = 1 + min(dp[i-1][j],
                           dp[i][j-1], dp[i-1][j-1])
return dp[m][n]
Palindrome DP — LC 516
Longest Palindromic Subsequence
Interval DP. dp[i][j] = LPS of s[i..j]. Outer chars match: +2. No match: max(shrink left, shrink right).
Word Break — LC 139
Can s be segmented into dict words?
dp[i] = can form s[:i]. Try all split points j. If dp[j] is True and s[j:i] is in dict, dp[i] = True.
Wildcard Matching — LC 44
? matches one char, * matches any
dp[i][j] = does s[:i] match p[:j]? The * case: dp[i-1][j] (match one more) or dp[i][j-1] (match empty).
07 — DP on trees & graphs
Tree DP uses post-order traversal — solve children first, then combine at the parent. Graph DP is memoised DFS.
Tree DP — House Robber III — LC 337
Return two values per node: (profit if robbed, profit if skipped).
from functools import cache

def rob(root):
    @cache
    def dp(node):
        if not node: return 0, 0  # (rob, skip)
        lr, ls = dp(node.left)
        rr, rs = dp(node.right)
        # rob this node: skip both children
        rob = node.val + ls + rs
        # skip this node: best of each child
        skip = max(lr,ls) + max(rr,rs)
        return rob, skip
    return max(dp(root))
Memo DFS — Longest Path — LC 329
Longest increasing path in a matrix. Memoised DFS on a DAG.
from functools import cache

def longest_path(matrix):
    m, n = len(matrix), len(matrix[0])

    @cache
    def dp(r, c):
        best = 1
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r+dr, c+dc
            if 0<=nr<m and 0<=nc<n:
                if matrix[nr][nc] > matrix[r][c]:
                    best = max(best, 1+dp(nr,nc))
        return best

    return max(dp(r,c) for r in range(m)
                        for c in range(n))
Tree diameter / path sum
LC 543, LC 124
At each node return the best contribution to parent. Update a global max with the best path through this node (left + right).
DP on DAG
Topological order = fill order
Process nodes in topological order. Memoised DFS handles this automatically — it solves dependencies before the node that needs them.
Bitmask DP
State = which nodes visited (n <= 20)
dp[mask][i] = best answer having visited the set "mask", ending at node i. Used for TSP and assignment problems.
08 — Space optimisation
Most 2D DP only looks at the current and previous row. Reduce O(nm) to O(m) space by keeping just two rows — or even two variables.
Rolling two rows — LCS 
# LCS: O(n*m) space → O(m) space
def lcs_opt(s1, s2):
    m, n = len(s1), len(s2)
    prev = [0] * (n+1)
    for i in range(1, m+1):
        curr = [0] * (n+1)
        for j in range(1, n+1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev = curr
    return prev[n]
Two variables — Fibonacci-type 
# When dp[i] only needs dp[i-1] and dp[i-2]
# use two variables, not an array

# House Robber — O(1) space:
prev2, prev1 = 0, 0
for num in nums:
    prev2, prev1 = prev1, max(prev1, prev2+num)
return prev1

# Fibonacci — O(1) space:
a, b = 0, 1
for _ in range(n): a, b = b, a+b
return a

The knapsack direction rule: 0/1 knapsack — iterate backwards (right to left). Each item used at most once. Unbounded knapsack — iterate forwards (left to right). Items can be reused. This single loop direction is the entire difference between the two variants in 1D.

10 — Interview approach
How to tackle a DP problem in an interview. Most people fail by jumping to the optimal solution before understanding the problem.
Step-by-step approach
1
Recognise it is DP. Keywords: minimum, maximum, number of ways, can you reach, longest, shortest. Multiple choices at each step with no obvious greedy rule.
2
Write brute force recursion first. Do not jump to DP. Say "this naive recursion has overlapping subproblems so I will add memoisation."
3
Add @cache. Wrap the recursion. This IS already DP. Verify it gives correct answers on examples before optimising.
4
Convert to tabulation (if asked). Fill a table from smallest subproblem to largest. Handle base cases explicitly.
5
Optimise space (if asked). Can you reduce to 1-2 rows? Only do this after the solution is verified correct.
Common mistakes
What causes wrong answers
Wrong table size (need n+1 not n). Wrong base case (dp[0]=0 vs dp[0]=1). Wrong loop direction in knapsack. Negative index in Python.
Recognising DP problems
Keywords and shapes
Decision at each step. Optimal answer depends on sub-answers. Asking for count, min, max, or yes/no. Brute force is exponential.
Top-down vs bottom-up
Which to use in interviews
Top-down is easier to write and harder to get wrong. Start there. Only convert to bottom-up if asked or if you need space optimisation.
Reconstructing the path
Not just the value — the actual choices
Keep the full dp table. Trace back from dp[n] following the decisions that were made. Backward pass through the table.
Complexity analysis
Time = states x work per state
Count distinct states then multiply by work per state. 1D: O(n). 2D string: O(mn). Knapsack: O(nW). Interval: O(n³). Bitmask: O(2^n · n).
