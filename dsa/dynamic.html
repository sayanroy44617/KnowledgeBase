Dynamic Programming

DP is just recursion with memory. Store results you have already computed so you never solve the same subproblem twice. Every pattern below follows the same 4 steps — once you see the skeleton, the code almost writes itself.

01 — What is Dynamic Programming?

One-line definition: DP = recursion + a cache. If your recursive function ever calls itself with the same input twice, store the result the first time and look it up the second time. That is literally it.

Property 1
Overlapping subproblems
The same subproblem appears in multiple branches of your recursion. fib(5) needs fib(3). fib(4) also needs fib(3). Without a cache — repeated work. Click to see the call tree.
Property 2
Optimal substructure
The best answer to the big problem is built from the best answers to smaller problems. Shortest path A→C through B needs shortest A→B plus shortest B→C.
Two approaches
Top-down vs Bottom-up
Top-down: write recursion, add a HashMap cache. Bottom-up: fill an array from small to large. Same answer, different direction.
DP vs Greedy
When greedy is not enough
Greedy picks one local best and never looks back. DP tries ALL choices and keeps the global best. If you can find one example where greedy fails — use DP.
Why it matters
O(2^n) → O(n) with one change
fib(50) without cache: 2^50 calls (takes forever). With cache: 50 calls. Adding a HashMap is the entire difference.
Fibonacci without cache — see the wasted work 
                    fib(5)
               /           \
           fib(4)           fib(3)  ← computed AGAIN below
          /     \          /     \
       fib(3)  fib(2)   fib(2)  fib(1)
       /   \   /   \    /   \
    fib(2) fib(1) ...  ...  ...

  fib(3) computed 2x   fib(2) computed 3x   fib(1) computed 5x
  For n=50: ~2^50 calls. A HashMap makes each computed ONCE.
02 — The 4-step framework (works for every DP problem)
Do not try to jump straight to the solution. Follow these 4 steps every time and DP becomes systematic.
The universal DP recipe
1
Define what dp[i] means in English. "dp[i] is the maximum money I can rob from the first i houses." If you cannot say it in one sentence, your state is wrong.
2
Write the recurrence. What choices do I have at step i? For each choice, express the answer using already-solved subproblems. Take the best (Math.max / Math.min) or sum them.
3
Write the base cases. What is the answer for the smallest input? dp[0] = 0? dp[1] = nums[0]? Wrong base cases give wrong answers silently — test them manually first.
4
Find where the answer lives. Often dp[n]. Sometimes Math.max of the whole dp array. Sometimes dp[m][n]. Always ask: after filling the table, which cell is my answer?
Top-down — HashMap cache
Start with plain recursion. Add a HashMap. Done — that IS DP.
int[] memo;

int fib(int n) {
    // base cases
    if (n <= 1) return n;
    // already solved? return cached answer
    if (memo[n] != -1) return memo[n];
    // solve it, store it, return it
    memo[n] = fib(n - 1) + fib(n - 2);
    return memo[n];
}
// call: Arrays.fill(memo, -1); fib(n);
Bottom-up — fill array left to right
No recursion. Fill dp[] from smallest subproblem to largest.
int fib(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    // base cases
    dp[0] = 0;
    dp[1] = 1;
    // fill smallest → largest
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2]; // recurrence
    }
    return dp[n]; // answer
}

Which to use? In an interview, start with top-down — it is easier to write and harder to mess up. Convert to bottom-up only if asked, or if you need to optimise space.

02b — The most important insight: Recursion → Table (Aditya Verma method)

The secret that makes DP click: Tabulation is not a "different algorithm." It is your recursive solution with function calls replaced by table lookups. The logic inside the if/else is identical. The only change is n-1 becomes i-1 (a row above in the table) and a function call becomes an array index.

Step 1 — Write the recursion first
Always start here. Write the naive recursive solution with a clear choice diagram: include the item OR exclude it. Do not think about DP yet.
// Subset Sum — recursive version
// "Can we pick from arr[0..n-1] to hit target?"
boolean solve(int[] arr, int n, int target) {
    // BASE CASES
    if (target == 0) return true;  // sum reached!
    if (n == 0) return false;      // no items left

    // CHOICE: can we include arr[n-1]?
    if (arr[n-1] <= target) {
        // INCLUDE: use item, reduce target
        // EXCLUDE: skip item, keep target
        return solve(arr, n-1, target-arr[n-1])
            || solve(arr, n-1, target);
    } else {
        // MUST EXCLUDE: item is too big
        return solve(arr, n-1, target);
    }
}
Step 2 — Add memo (top-down DP)
The recursion above recomputes the same (n, target) pairs many times. Add a 2D memo array. Check it before computing, store before returning.
// Add memo — ZERO code logic changes
int[][] memo = new int[n+1][sum+1];
// initialise all to -1 (unvisited)
for (int[] row : memo) Arrays.fill(row, -1);

boolean solve(int[] arr, int n, int target) {
    if (target == 0) return true;
    if (n == 0) return false;
    // CHECK: already solved this (n, target)?
    if (memo[n][target] != -1)
        return memo[n][target] == 1;
    boolean result;
    if (arr[n-1] <= target)
        result = solve(arr,n-1,target-arr[n-1])
              || solve(arr,n-1,target);
    else
        result = solve(arr,n-1,target);
    // STORE before returning
    memo[n][target] = result ? 1 : 0;
    return result;
}
The transformation — this is what Aditya Verma is teaching
Every recursive call solve(arr, n-1, ...) becomes a table lookup t[i-1][...]. The logic inside the if/else does not change at all.
RECURSIVE CALL
if (arr[n-1] <= target) {

    result =
      solve(arr, n-1, target-arr[n-1])
   || solve(arr, n-1, target);

} else {

    result = solve(arr, n-1, target);

}
TABLE LOOKUP — same logic, no recursion
if (arr[i-1] <= j) {

    t[i][j] =
      t[i-1][j - arr[i-1]]
   || t[i-1][j];

} else {

    t[i][j] = t[i-1][j];

}

target → j

Step 3 — Full bottom-up tabulation (the end result)
Build the table row by row. Row 0 = base cases. Every row i uses row i-1, which is already filled. No recursion, no stack.
boolean subsetSum(int[] arr, int sum) {
    int n = arr.length;
    // t[i][j] = "can we make sum j using first i items?"
    boolean[][] t = new boolean[n+1][sum+1];

    // BASE CASE 1: sum==0 is always true (take nothing)
    for (int i = 0; i <= n; i++) t[i][0] = true;
    // BASE CASE 2: n==0, sum!=0 is always false (default)
    // (boolean array is false by default — nothing to do)

    // FILL the table row by row (small → large)
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= sum; j++) {
            if (arr[i-1] <= j)
                // SAME logic as recursion — just table lookup
                t[i][j] = t[i-1][j - arr[i-1]] || t[i-1][j];
            else
                t[i][j] = t[i-1][j]; // item too big, exclude
        }
    }
    return t[n][sum]; // answer: can we make full sum?
}
What the table looks like — arr=[2,3,7], sum=5
i\j 0 1 2 3 4 5
0 (no items) T F F F F F
1 (item=2) T F T F F F
2 (item=3) T F T T F T
3 (item=7) T F T T F T
How to read this: t[2][5] = T means "using the first 2 items (2 and 3), we CAN make sum 5 (2+3=5)". t[1][3] = F means "using only item 2, we CANNOT make sum 3". The answer is always in the bottom-right cell: t[n][sum].
How t[2][5] = T was computed: arr[1]=3, j=5. Since 3 <= 5: t[2][5] = t[1][5-3] || t[1][5] = t[1][2] || t[1][5] = T || F = T. That T in t[1][2] comes from row above — item 2 makes sum 2. This is i-1 in action.
The Aditya Verma process in 3 steps
1
Write the recursive Choice Diagram. Just handle include vs exclude. Get it working. Do not think about efficiency yet.
2
Identify the changing variables. In the recursion, n and target change with each call. These become your table dimensions: t[n+1][sum+1].
3
Copy the if/else into a nested for loop. Replace solve(n-1, ...) with t[i-1][...]. Fill base cases first. That is the entire conversion.
Why i-1 means "previous item"
In the table, row i represents the decision about item i. Row i-1 is the world before item i was considered — which is exactly what solve(n-1, ...) means in the recursion.

So t[i-1][j] = "best answer not using item i" = solve(n-1, target)

And t[i-1][j-arr[i-1]] = "best answer with item i subtracted" = solve(n-1, target-arr[n-1])

Once you see this, you can write tabulation in 2 minutes: Get the recursion right. The table size = dimensions of your recursive parameters + 1. The if/else inside the loops = copy from the recursion. Base cases = your recursive base cases written as table initialisations. The answer = t[n][sum] (last values of your parameters).

04 — 1D DP patterns
dp is a 1D array. Each cell uses one or two previous cells. These are the building blocks — get comfortable here before moving to 2D.
Pattern: Fibonacci-type
LC 70 — Climbing Stairs
Plain English: dp[i] = number of ways to reach step i. You can reach step i from step i-1 (take 1 step) or from step i-2 (take 2 steps).
// dp[i] = dp[i-1] + dp[i-2]
int climbStairs(int n) {
    if (n <= 2) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1; // 1 way to reach step 1
    dp[2] = 2; // 2 ways to reach step 2
    for (int i = 3; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}
Pattern: Include or Exclude
LC 198 — House Robber
Plain English: dp[i] = max money from first i houses. At each house: rob it (skip the previous) OR skip it (keep the previous best).
// At house i: rob i + dp[i-2]  OR  skip → dp[i-1]
int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    int[] dp = new int[n];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);
    for (int i = 2; i < n; i++)
        dp[i] = Math.max(dp[i-1], nums[i] + dp[i-2]);
    return dp[n - 1];
}
Pattern: Extend or Restart
LC 53 — Maximum Subarray (Kadane)
Plain English: dp[i] = max subarray sum ending exactly at index i. Either extend the previous subarray or throw it away and start fresh.
// dp[i] = max(nums[i], dp[i-1] + nums[i])
// "start fresh"  OR  "extend previous"
int maxSubArray(int[] nums) {
    int curr = nums[0], best = nums[0];
    for (int i = 1; i < nums.length; i++) {
        curr = Math.max(nums[i], curr + nums[i]);
        best = Math.max(best, curr);
    }
    return best;
}
Pattern: Try Every Option
LC 322 — Coin Change
Plain English: dp[a] = fewest coins to make amount a. For each amount, try every coin — pick whichever gives the minimum.
// dp[a] = min coins for amount a
int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1); // "infinity"
    dp[0] = 0;                   // 0 coins to make 0
    for (int a = 1; a <= amount; a++)
        for (int coin : coins)
            if (coin <= a)
                dp[a] = Math.min(dp[a], dp[a-coin]+1);
    return dp[amount] > amount ? -1 : dp[amount];
}
Pattern: Look Back at Everything
LC 300 — Longest Increasing Subsequence (LIS)
Plain English: dp[i] = length of the longest increasing subsequence that ends at index i. For each i, look back at every j before it — if nums[j] < nums[i], we can extend the LIS ending at j.
int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1); // every element alone = LIS of 1
    int best = 1;
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i])              // can extend?
                dp[i] = Math.max(dp[i], dp[j]+1);
        }
        best = Math.max(best, dp[i]);
    }
    return best;
}
05 — 2D DP patterns
dp is a 2D grid. The state dp[i][j] usually means: "the best answer considering the first i items from one input and the first j items from another."
Grid: Unique Paths — LC 62
Plain English: dp[i][j] = number of ways to reach cell (i,j) moving only right or down. You can only arrive from the cell above or the cell to the left.
// dp[i][j] = dp[i-1][j] + dp[i][j-1]
int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    // base cases: first row and first col = 1 way
    for (int i = 0; i < m; i++) dp[i][0] = 1;
    for (int j = 0; j < n; j++) dp[0][j] = 1;
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = dp[i-1][j] + dp[i][j-1];
    return dp[m-1][n-1]; // answer
}
Grid: Min Path Sum — LC 64
Plain English: dp[i][j] = minimum cost to reach cell (i,j). The cost is the current cell plus whichever neighbour (above or left) was cheaper to reach.
// dp[i][j] = grid[i][j] + min(above, left)
int minPathSum(int[][] g) {
    int m = g.length, n = g[0].length;
    int[][] dp = new int[m][n];
    dp[0][0] = g[0][0];
    for (int i=1;i<m;i++) dp[i][0]=dp[i-1][0]+g[i][0];
    for (int j=1;j<n;j++) dp[0][j]=dp[0][j-1]+g[0][j];
    for (int i=1;i<m;i++)
        for (int j=1;j<n;j++)
            dp[i][j]=g[i][j]+Math.min(dp[i-1][j],dp[i][j-1]);
    return dp[m-1][n-1];
}
Interval DP
dp[i][j] = answer for range i..j
Fill by increasing length. Try every split point k inside [i,j]. Burst Balloons, Palindromic Subsequence. O(n³).
What dp[i][j] means
Common 2D state interpretations
String problems: first i chars of s1, first j of s2. Knapsack: first i items, capacity j. Grid: row i, col j. Interval: range i to j.
Grid DP tips
Obstacles, reverse fill, base cases
Obstacles: set dp[i][j]=0 at blocked cell. Reverse fill: Dungeon Game fills bottom-right to top-left. First row and col are always base cases.
06 — Knapsack patterns
The most important DP family. The core idea: for each item, decide whether to include it or exclude it. Include it and reduce your remaining capacity. Exclude it and move on.

The two variants: 0/1 knapsack — each item usable at most once (iterate capacity backwards). Unbounded knapsack — each item usable unlimited times (iterate capacity forwards). One loop direction is the only difference in the 1D version.

0/1 Knapsack — each item once
Plain English: dp[w] = max value you can pack into capacity w. For each item, go backwards through capacities so you never use the same item twice.
// Optimised 1D version — iterate BACKWARDS
int knapsack01(int[] wt, int[] val, int W) {
    int[] dp = new int[W + 1];
    for (int i = 0; i < wt.length; i++) {
        // go RIGHT to LEFT — prevents reusing item i
        for (int w = W; w >= wt[i]; w--) {
            dp[w] = Math.max(dp[w],
                         val[i] + dp[w - wt[i]]);
        }
    }
    return dp[W];
}
Unbounded Knapsack — reuse items
Plain English: Same idea but you can take each item multiple times. Iterate capacity forwards so the same item can be used again.
// Coin Change is unbounded knapsack
int unboundedKnapsack(int[] wt, int[] val, int W) {
    int[] dp = new int[W + 1];
    for (int i = 0; i < wt.length; i++) {
        // go LEFT to RIGHT — allows reusing item i
        for (int w = wt[i]; w <= W; w++) {
            dp[w] = Math.max(dp[w],
                         val[i] + dp[w - wt[i]]);
        }
    }
    return dp[W];
}
Subset Sum — LC 416
Can we pick elements that sum to target?
// LC 416 — Equal Partition
// dp[t] = can we make sum t from the array?
boolean canPartition(int[] nums) {
    int sum = 0;
    for (int n : nums) sum += n;
    if (sum % 2 != 0) return false;
    int target = sum / 2;
    boolean[] dp = new boolean[target + 1];
    dp[0] = true; // empty set makes sum 0
    for (int num : nums)
        for (int t = target; t >= num; t--)
            dp[t] = dp[t] || dp[t - num];
    return dp[target];
}
Coin Change II — LC 518
Count combinations (loop order matters!)
// dp[a] = number of ways to make amount a
// coins outer loop = combinations (not permutations)
int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1; // 1 way to make 0: use nothing
    for (int coin : coins)           // coins OUTSIDE
        for (int a = coin; a <= amount; a++)
            dp[a] += dp[a - coin];
    return dp[amount];
}
07 — String DP
Almost always a 2D table. dp[i][j] involves the first i characters of s1 and the first j characters of s2. The recurrence always splits on one question: do the current characters match?
LCS — Longest Common Subsequence
Plain English: dp[i][j] = length of the longest common subsequence of s1[:i] and s2[:j]. If the current characters match, extend. If not, skip one character from either string.
// match → dp[i-1][j-1] + 1
// no match → max(skip s1 char, skip s2 char)
int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m+1][n+1];
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i-1) == s2.charAt(j-1))
                dp[i][j] = dp[i-1][j-1] + 1; // match!
            else
                dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}
Edit Distance — LC 72
Plain English: dp[i][j] = minimum edits to turn s1[:i] into s2[:j]. If chars match: free. If not: 1 + the best of (insert, delete, replace).
// match → dp[i-1][j-1]  (no edit needed)
// no match → 1 + min(insert, delete, replace)
int minDistance(String s1, String s2) {
    int m=s1.length(), n=s2.length();
    int[][] dp = new int[m+1][n+1];
    // base: convert to/from empty string
    for(int i=0;i<=m;i++) dp[i][0]=i;
    for(int j=0;j<=n;j++) dp[0][j]=j;
    for(int i=1;i<=m;i++)
        for(int j=1;j<=n;j++){
            if(s1.charAt(i-1)==s2.charAt(j-1))
                dp[i][j]=dp[i-1][j-1];
            else
                dp[i][j]=1+Math.min(dp[i-1][j],
                             Math.min(dp[i][j-1],dp[i-1][j-1]));
        }
    return dp[m][n];
}
Palindrome — LC 516
Longest Palindromic Subsequence
Interval DP. dp[i][j] = LPS of s[i..j]. Outer chars match: dp[i+1][j-1] + 2. No match: max of shrinking either end.
Word Break — LC 139
Can s be split into dictionary words?
dp[i] = can we form s[0..i]? Try every split point j — if dp[j] is true and s[j..i] is a word, dp[i] = true.
Wildcard — LC 44
? matches one char, * matches any
dp[i][j] = does s[:i] match p[:j]? The * case: match one more char (dp[i-1][j]) OR match empty (dp[i][j-1]).
08 — DP on Trees
Tree DP uses post-order traversal — solve children first, then combine results at the parent. The function returns one or more values per node.
Tree DP — House Robber III — LC 337
Plain English: At each node, return a pair: (profit if we rob this node, profit if we skip this node). Then the parent decides which option is better.
// returns int[] {robThisNode, skipThisNode}
int[] dp(TreeNode node) {
    if (node == null) return new int[]{0, 0};

    int[] left  = dp(node.left);
    int[] right = dp(node.right);

    // rob this node: must skip both children
    int rob  = node.val + left[1] + right[1];
    // skip this node: take best from each child
    int skip = Math.max(left[0],left[1])
             + Math.max(right[0],right[1]);

    return new int[]{rob, skip};
}

int rob(TreeNode root) {
    int[] res = dp(root);
    return Math.max(res[0], res[1]);
}
Memo DFS — Longest Inc. Path — LC 329
Plain English: dp[r][c] = longest increasing path starting from cell (r,c). DFS + memo — if we already know the answer for (r,c), return it instantly.
int[][] memo;
int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

int dfs(int[][] matrix, int r, int c) {
    if (memo[r][c] != 0) return memo[r][c]; // cached!
    int best = 1;
    for (int[] d : dirs) {
        int nr = r+d[0], nc = c+d[1];
        if (nr>=0 && nr<matrix.length
         && nc>=0 && nc<matrix[0].length
         && matrix[nr][nc] > matrix[r][c])
            best = Math.max(best, 1+dfs(matrix,nr,nc));
    }
    memo[r][c] = best;
    return best;
}
Tree diameter / path sum
LC 543, LC 124
Return best contribution to parent. Update a global max with left path + right path through this node. One global variable does the trick.
DP on DAG
Memo DFS = topological order
Memo DFS automatically processes dependencies before the node that needs them. You get topological order for free without explicitly sorting.
Bitmask DP
State = which nodes visited (n <= 20)
dp[mask][i] = best answer having visited the set encoded in mask, currently at node i. Used for TSP and assignment problems.
09 — Space Optimisation
Most 2D DP tables only ever look at the current row and the previous row. You can reduce O(n*m) space down to O(m) by keeping just those two rows — or even just two variables.
Rolling rows — LCS
Only keep the previous row. At the start of each outer loop, swap prev into curr.
// LCS: O(n*m) space → O(n) space
int lcsOpt(String s1, String s2) {
    int m=s1.length(), n=s2.length();
    int[] prev = new int[n+1];
    for (int i=1;i<=m;i++){
        int[] curr = new int[n+1];
        for (int j=1;j<=n;j++){
            if(s1.charAt(i-1)==s2.charAt(j-1))
                curr[j] = prev[j-1] + 1;
            else
                curr[j] = Math.max(prev[j],curr[j-1]);
        }
        prev = curr;
    }
    return prev[n];
}
Two variables — Fibonacci-type
When dp[i] only needs dp[i-1] and dp[i-2], two int variables replace the whole array.
// House Robber — O(1) space
int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;
    for (int num : nums) {
        int curr = Math.max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// Fibonacci — O(1) space
int fib(int n) {
    int a = 0, b = 1;
    for (int i = 0; i < n; i++) {
        int tmp = a + b; a = b; b = tmp;
    }
    return a;
}

The knapsack direction rule (critical!): 0/1 knapsack — iterate capacity right to left (W down to weight[i]). This prevents reusing the same item. Unbounded knapsack — iterate capacity left to right (weight[i] up to W). This allows reusing items. One direction = one rule change.

11 — Interview approach
Most people fail DP interviews by jumping to the optimal solution too fast. The steps below work every time.
The 5-step interview playbook
1
Spot that it is DP. Keywords: minimum, maximum, number of ways, can you reach, longest, shortest. Multiple choices at each step. No obvious single best pick at each step.
2
Write brute force recursion first. Solve the problem naively with recursion and no cache. Say aloud: "this has overlapping subproblems so I will memoize it."
3
Add int[] memo and check it first. Before computing, check if memo[i] != -1. Store the result before returning. This is already DP. Verify on a small example.
4
Convert to bottom-up if asked. Fill an int[] dp array using a for loop in the order from smallest subproblem to largest. Handle base cases before the loop.
5
Optimise space if asked. Can you keep only the last 1-2 rows? Only do this after the solution is correct and verified.
Common mistakes
What causes wrong answers
Array of size n instead of n+1. Wrong base case (0 vs 1 for count problems). Iterating forwards in 0/1 knapsack (must go backwards). Forgetting to check if memo is filled.
Recognising DP
Keywords and shapes
Brute force is exponential. Same subproblem appears in multiple branches. Asking for count, min, max, or true/false with "best possible" framing.
Top-down vs bottom-up
Which to use when
Top-down: write recursion, add memo[], done. Use this first. Bottom-up: fill dp[] with a loop. Needed for space optimisation or when interviewer asks iterative.
Reconstructing the path
Not just the value — the actual choices
Keep the full 2D table. Walk backwards from dp[n] to dp[0], checking which choice produced the current cell value. This is how you print the actual LCS or selected items.
Complexity analysis
Time = states x work per state
Count distinct dp states, multiply by work per state. 1D: O(n). 2D string: O(mn). Knapsack: O(nW). Interval DP: O(n³). Bitmask: O(2^n * n).
