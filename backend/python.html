01 — Data types & builtins
Python's built-in types and the operations that matter. Know the time complexity of each operation — it comes up in code reviews and interviews.
list
Ordered, mutable sequence
O(1) append/pop from end. O(n) insert/delete at index. O(n) search. Dynamic array underneath.
dict
Hash map — O(1) average
Insertion-ordered since Python 3.7. O(1) get/set/delete average. O(n) worst case (hash collision).
set / frozenset
Hash set — unique elements
O(1) add/remove/in. O(n) union/intersection. frozenset is immutable and hashable — can be a dict key.
tuple
Immutable sequence
Hashable (if elements are). Used as dict keys, namedtuple for readable records. Faster than list for iteration.
str
Immutable sequence of chars
f-strings, join, split, strip, replace, startswith/endswith, encode/decode. Strings are interned for short literals.
Essential builtins
map, filter, zip, enumerate
All return lazy iterators. zip stops at shortest. enumerate adds index. map/filter replaced by comprehensions in modern Python.
Unpacking & *args/**kwargs
Starred expressions
a, *b, c = [1,2,3,4,5]. Spread iterables into function calls. Merge dicts with {**d1, **d2}.

02 — Comprehensions & expressions
Python's most expressive feature. Prefer comprehensions over map/filter for readability. Use generator expressions to avoid materialising large lists.
List / dict / set comprehensions
# list comprehension
squares = [x**2 for x in range(10) if x % 2 == 0]
 
# dict comprehension
inv = {v: k for k, v in d.items()}
 
# set comprehension
unique = {x.lower() for x in words}
 
# nested — flatten a 2D list
flat = [x for row in matrix for x in row]
 
# conditional expression (ternary)
label = "even" if n % 2 == 0 else "odd"
Generator expressions
# generator expression — lazy, no list created
total = sum(x**2 for x in range(1_000_000))
 
# any / all with generator
any(x > 100 for x in data)
all(isinstance(x, int) for x in data)
 
# walrus operator := (Python 3.8+)
results = [y for x in data if (y := transform(x)) is not None]
 
# generator as function argument
csv_line = ",".join(str(v) for v in row)
Walrus operator :=
Assign and test in one step
Avoid computing the same value twice. Useful in while loops, comprehension filters, and match expressions.
Sorting & key functions
sorted, sort, key=, operator
key= takes a function applied to each element. operator.itemgetter and attrgetter for clean key functions. Stable sort.
Slicing & step
a[start:stop:step]
a[::-1] reverses. a[::2] every other. a[:n] first n. Slices create copies. slice() object for reusable slices.
03 — OOP & magic methods
Python's object model is protocol-based. Magic methods (dunder methods) let your classes integrate with Python's built-in syntax and operators.
Class essentials
class Vector:
def __init__(self, x: float, y: float):
self.x = x; self.y = y
 
# readable repr
def __repr__(self) -> str:
return f"Vector({self.x}, {self.y})"
 
# addition operator
def __add__(self, other) -> "Vector":
return Vector(self.x + other.x, self.y + other.y)
 
# equality
def __eq__(self, other) -> bool:
return (self.x, self.y) == (other.x, other.y)
 
# make hashable
def __hash__(self) -> int:
return hash((self.x, self.y))
Class methods & properties
class Temperature:
def __init__(self, celsius: float):
self._celsius = celsius
 
# classmethod — alternative constructor
@classmethod
def from_fahrenheit(cls, f: float) -> "Temperature":
return cls((f - 32) * 5 / 9)
 
# property — computed attribute
@property
def fahrenheit(self) -> float:
return self._celsius * 9/5 + 32
 
# staticmethod — no self or cls
@staticmethod
def is_valid(c: float) -> bool:
return c >= -273.15
dataclass
Auto-generate boilerplate
@dataclass generates __init__, __repr__, __eq__. frozen=True adds __hash__. field() for defaults. slots=True for memory efficiency.
Magic methods reference
The full dunder protocol
__len__, __getitem__, __iter__, __contains__, __enter__/__exit__, __call__, __slots__, __class_getitem__.
Inheritance & MRO
super(), ABC, MRO
super() without args in Python 3. Multiple inheritance resolution via C3 linearisation. ABC for abstract base classes.
Context managers
__enter__ / __exit__ / contextlib
with statement. Custom via __enter__/__exit__ or @contextmanager. suppress(), ExitStack for multiple resources.
__slots__
Memory optimisation
Prevents __dict__ creation per instance. Reduces memory by 40-50% for classes with many instances. Loses dynamic attribute setting.
04 — Decorators & closures
Decorators are just functions that take a function and return a function. Understanding closures is the key to understanding decorators.
Closure & decorator anatomy
# closure — function that captures outer scope
def make_multiplier(n: int):
def multiply(x: int) -> int:
return x * n # n captured from outer
return multiply
 
# basic decorator
from functools import wraps
def log_call(func):
@wraps(func) # preserve metadata
def wrapper(*args, **kwargs):
print(f"Calling {func.__name__}")
result = func(*args, **kwargs)
print(f"Done")
return result
return wrapper
return wrapper
Decorator with arguments
# decorator factory (takes args)
def retry(times: int = 3, exceptions=(Exception,)):
def decorator(func):
@wraps(func)
def wrapper(*args, **kwargs):
for attempt in range(times):
try:
return func(*args, **kwargs)
except exceptions:
if attempt == times - 1:
raise
return wrapper
return decorator
return decorator
 
@retry(times=3, exceptions=(ConnectionError,))
def fetch_data(url: str): ...
functools.cache / lru_cache
Memoisation in one line
@cache (unbounded) or @lru_cache(maxsize=128). Cache function results. cache_info() and cache_clear(). Works with any hashable args.
Class-based decorators
__call__ on a class
A class with __call__ can be used as a decorator. Useful when you need state across calls (rate limiter, call counter).
Stacking decorators
Order matters
Applied bottom-up. @A @B def f → A(B(f)). @wraps is critical — without it, __name__, __doc__, and type hints are lost.
05 — Generators & iterators
Generators produce values lazily — one at a time, on demand. They are the backbone of memory-efficient data processing in Python.
Generator functions
# yield pauses execution, resumes on next()
def infinite_counter(start: int = 0):
while True:
yield start
start += 1
 
# yield from — delegate to sub-generator
def chain(*iterables):
for it in iterables:
yield from it
 
# generator pipeline
lines = (line.strip() for line in open("file.txt"))
non_empty = (line for line in lines if line)
parsed = (json.loads(line) for line in non_empty)
Custom iterators
# implement __iter__ and __next__
class Range:
def __init__(self, n): self.n = n; self.i = 0
def __iter__(self): return self
def __next__(self):
if self.i >= self.n: raise StopIteration
val = self.i; self.i += 1; return val
 
# send() — two-way communication
def accumulator():
total = 0
while True:
value = yield total
if value is not None: total += value
itertools
The iterator toolkit
chain, islice, product, combinations, permutations, groupby, cycle, repeat, accumulate, takewhile, dropwhile.
Generator patterns
Pipelines & streaming
Process large files line-by-line. Build ETL pipelines where each stage is a generator. Avoid loading everything into memory.
yield from
Sub-generator delegation
Delegates to a sub-generator. Propagates send() and throw(). Simpler than manually forwarding values. Used in async internally.
06 — Type hints & generics
Python 3.10+ has dramatically improved type hint syntax. Use them for IDE support, documentation, and runtime validation with Pydantic.
Modern type hint syntax
from __future__ import annotations # defer eval
from typing import TypeVar, Generic, Protocol
 
# built-in generics (3.9+)
def process(items: list[int]) -> dict[str, int]: ...
 
# union types (3.10+)
def parse(val: str | int | None) -> float: ...
 
# TypeAlias (3.10+)
Vector = list[float]
 
# Optional shorthand
name: str | None = None # same as Optional[str]
 
# Callable
from typing import Callable
Handler = Callable[[str, int], None]
Generics & TypeVar
# TypeVar for generic functions
T = TypeVar("T")
def first(items: list[T]) -> T:
return items[0]
 
# Generic class
class Stack(Generic[T]):
def __init__(self): self._items: list[T] = []
def push(self, item: T) -> None: ...
def pop(self) -> T: ...
 
# Protocol — structural subtyping (duck typing)
class Drawable(Protocol):
def draw(self) -> None: ...
 
# ParamSpec for decorator typing (3.10+)
from typing import ParamSpec, Concatenate
P = ParamSpec("P")
Literal, Final, TypedDict
Precise type constraints
Literal["GET","POST"] for exact string values. Final for constants. TypedDict for typed dict shapes. Annotated for metadata.
Type narrowing
isinstance, assert, match
isinstance() narrows union types. assert narrows Optional. match/case for structural pattern matching. TypeGuard for custom narrowing.
Types + Pydantic
Runtime validation from hints
Pydantic v2 uses Python type hints for runtime validation and serialisation. Field, validator, model_validator, computed_field.
07 — Async / asyncio
Python's async model is single-threaded cooperative multitasking. async/await lets you write concurrent code that looks synchronous. Critical for FastAPI.
async / await fundamentals
# coroutine — defined with async def
async def fetch(url: str) -> str:
async with httpx.AsyncClient() as client:
resp = await client.get(url)
return resp.text
 
# run multiple coroutines concurrently
async def main():
results = await asyncio.gather(
fetch("https://api1.com"),
fetch("https://api2.com"),
return_exceptions=True
)
 
# create tasks (fire and forget)
task = asyncio.create_task(some_coroutine())
await task
Async patterns
# async context manager
async with asyncpg.create_pool(...) as pool:
row = await pool.fetchrow("SELECT ...")
 
# async generator
async def stream_rows(db):
async for row in db.cursor("SELECT ..."):
yield row
 
# timeout
async with asyncio.timeout(5.0):
result = await slow_operation()
 
# run sync code in thread pool
result = await asyncio.to_thread(blocking_fn, arg)
 
# TaskGroup (3.11+) — structured concurrency
async with asyncio.TaskGroup() as tg:
t1 = tg.create_task(coro1())
t2 = tg.create_task(coro2())
Event loop & tasks
How asyncio actually works
Single-threaded event loop. await yields control back to the loop. CPU-bound work blocks — use ProcessPoolExecutor or to_thread.
async vs sync — when to use
IO-bound vs CPU-bound
async for IO (HTTP, DB, files). ThreadPoolExecutor for blocking IO. ProcessPoolExecutor for CPU. Never await a CPU-bound task.
Common async pitfalls
Mistakes that break concurrency
Forgetting await. Calling sync IO in async code. Shared mutable state. Not using gather for concurrent tasks. Blocking the event loop.
08 — Error handling
Python exceptions are objects. Build a clean error hierarchy, use specific except clauses, and always clean up resources with finally or context managers.
Exception hierarchy & patterns
# custom exception hierarchy
class AppError(Exception):
def __init__(self, msg: str, code: int = 500):
super().__init__(msg); self.code = code
 
class NotFoundError(AppError):
def __init__(self, resource: str):
super().__init__(f"{resource} not found", 404)
 
# exception chaining
try:
db.query(...)
except DBError as e:
raise AppError("DB failed") from e
 
# suppress specific exceptions
from contextlib import suppress
with suppress(FileNotFoundError):
os.remove("file.txt")
try / except / else / finally
try:
result = risky_operation()
except (ValueError, TypeError) as e:
# handle multiple types
logger.error("Bad input: %s", e)
except Exception:
# catch-all — use sparingly
raise # always re-raise if you catch all
else:
# runs ONLY if no exception raised
save(result)
finally:
# always runs — cleanup here
cleanup()
 
# ExceptionGroup (3.11+)
try: ...
except* ValueError as eg:
for exc in eg.exceptions: ...
Exception best practices
What to catch, what to raise
Never catch Exception silently. Catch specific types. Raise early, fail loudly. Use else for success path. Chain with raise...from.
ExceptionGroup (3.11+)
Multiple concurrent exceptions
TaskGroup raises ExceptionGroup when multiple tasks fail. except* syntax to handle groups by type. New in 3.11.
warnings module
DeprecationWarning & filters
warnings.warn() for deprecation notices. filterwarnings() to control display. Different from exceptions — don't stop execution.

collections — quick reference
from collections import defaultdict, Counter, deque
 
# defaultdict — no KeyError
graph = defaultdict(list)
graph["a"].append("b") # no init needed
 
# Counter — count anything
c = Counter("banana") # {"a":3,"n":2,"b":1}
c.most_common(2) # [("a",3),("n",2)]
c1 + c2 # merge counters
 
# deque — O(1) both ends
dq = deque(maxlen=3) # fixed-size sliding window
dq.appendleft(x) # O(1)
dq.rotate(1) # rotate right by 1
functools — quick reference
from functools import partial, reduce, cache, singledispatch
 
# partial — fix some arguments
double = partial(pow, exp=2) # pow(x, exp=2)
 
# reduce — fold left
product = reduce(lambda a, b: a * b, [1,2,3,4])
 
# cache — unbounded memoisation
@cache
def fib(n): return n if n < 2 else fib(n-1) + fib(n-2)
 
# singledispatch — method overloading by type
@singledispatch
def process(val): raise NotImplementedError
@process.register(int)
def _(val): return val * 2
10 — Pythonic patterns & interview topics
Patterns that distinguish intermediate Python from advanced Python. Things that come up in code reviews and technical interviews.
match / case (3.10+)
Structural pattern matching
Match on structure, not just values. Destructure dicts, lists, dataclasses. Guard clauses with if. Replaces isinstance chains.
Descriptors
How property works under the hood
__get__, __set__, __delete__. Data vs non-data descriptors. This is how @property, @staticmethod, @classmethod are implemented.
Metaclasses
Classes that create classes
type is the metaclass of all classes. __init_subclass__ is the modern alternative. Used by SQLAlchemy, Pydantic, Django ORM.
Memory & GC
Reference counting & cycles
CPython uses reference counting + cyclic GC. sys.getrefcount(), weakref for caches, __del__ pitfalls, tracemalloc for leaks.
Protocol vs ABC
Static vs runtime duck typing
Protocol = structural (no explicit inheritance). ABC = nominal (must inherit). Protocol is preferred for type hints; ABC for runtime isinstance checks.
The GIL
Global Interpreter Lock
Only one thread executes Python bytecode at a time. IO-bound: use threading. CPU-bound: use multiprocessing. GIL removed in 3.13 (opt-in).
