01 — Cluster Architecture
02 — Core Objects
Pod
Smallest deployable unit
1+ containers sharing network + storage. Ephemeral — never restart in place. Each gets a unique IP.
Deployment
Manages ReplicaSets
Declarative updates, rollback, scaling. Most common way to run stateless apps.
StatefulSet
Ordered, stable identities
Sticky pod names (pod-0, pod-1), persistent storage per pod. Use for DBs, Kafka, ZooKeeper.
DaemonSet
One pod per node
Runs on every node automatically. New nodes get it too. Use for log collectors, monitoring agents.
Job / CronJob
Run-to-completion
Job runs once and exits. CronJob runs on a schedule. Great for batch processing, DB migrations.
ConfigMap / Secret
Configuration injection
ConfigMap for plain config. Secret for sensitive data (base64 encoded — NOT encrypted by default).

04 — Storage
PersistentVolume
Admin-provisioned storage
Cluster-level resource. Has a reclaim policy: Retain, Recycle, Delete.
PersistentVolumeClaim
Pod's storage request
Requests size + access mode. Bound 1:1 to a PV. Lives in a namespace.
StorageClass
Dynamic provisioning
Auto-creates PVs on demand. Define provisioner + reclaim policy. Default class used if none specified.
emptyDir
Temp dir, lives with the pod. Deleted when pod dies. Good for scratch space.
hostPath
Mounts a file/dir from the host node. Avoid in prod — breaks node portability.
configMap / secret
Inject config files into pods as volumes. Read-only by default.

05 — Scheduling & Resources
Requests vs Limits
RequestGuaranteed minimum. Used for scheduling decisions.
LimitHard cap. OOMKilled if exceeded. Always set both.
resources:
requests:
cpu: "250m" memory: "128Mi"
limits:
cpu: "500m" memory: "256Mi"
Scheduling controls
nodeSelectorSimple label match to target node
nodeAffinityRequired or preferred rules, more expressive
TaintRepels pods from a node (applied to node)
TolerationPod explicitly ignores a taint
podAffinityCo-locate pods on same node/zone
06 — Health Probes
Liveness
Is it alive?
Fails → container is restarted. Use for deadlock detection. Don't set too aggressively.
Readiness
Ready for traffic?
Fails → removed from Service endpoints. Pod stays running. No traffic during warm-up.
Startup
Has it started yet?
Disables liveness + readiness until it passes. Prevents killing slow-starting apps.
Probe types
httpGet tcpSocket exec (command) grpc
initialDelaySecondsWait before first probe
periodSecondsHow often to probe
failureThresholdFailures before action taken
07 — RBAC
ServiceAccount
Identity for pods. Auto-mounted token at /var/run/secrets.
Role / ClusterRole
Set of permissions. Role = namespaced. ClusterRole = cluster-wide.
RoleBinding
Binds a Role to a subject (user, group, or ServiceAccount).
Namespace
Virtual cluster. Scope for names, RBAC, quotas, and network policies.
RBAC flow: ServiceAccount → RoleBinding → Role → permissions on resources (pods, secrets, etc.)

09 — Scaling & Availability
HPA
HorizontalPodAutoscaler
Auto-scales pod count based on CPU/memory or custom metrics. Requires metrics-server.
PDB
PodDisruptionBudget
Sets minAvailable or maxUnavailable. Protects during drains + rolling updates.
ResourceQuota
Namespace limits
Caps total CPU/memory/objects per namespace. LimitRange sets per-pod defaults.
VPA — VerticalPodAutoscaler
Automatically adjusts CPU/memory requests and limits. Three modes: Off (recommend only), Initial (set on creation), Auto (update running pods — causes restarts).
10 — Common Interview Questions
Pod lifecycle
Pending → Running → Succeeded / Failed / CrashLoopBackOff.

Know: init containers run first sequentially. postStart / preStop hooks. terminationGracePeriodSeconds (default 30s). SIGTERM → grace period → SIGKILL.
Rolling update
Controlled by maxSurge (extra pods during update) and maxUnavailable (pods that can go down). kubectl rollout undo reverts. History limited by revisionHistoryLimit.
QoS classes
Guaranteed — requests = limits.
Burstable — requests < limits.
BestEffort — no requests/limits set.

Eviction order: BestEffort → Burstable → Guaranteed.
Service discovery
kube-dns auto-creates DNS entries. svc.ns.svc.cluster.local. Env vars also injected into pods. kube-proxy manages iptables / IPVS rules for routing.
What happens when you create a pod?
kubectl → API Server → etcd (persists) → Scheduler picks node → API Server notifies kubelet → kubelet pulls image → Container Runtime starts container → kubelet reports status back.
CrashLoopBackOff
Pod keeps crashing and k8s is backing off restarts. Debug with: kubectl logs --previous, check liveness probe config, check resource limits (OOMKill), check entrypoint command.
Deployment vs StatefulSet — when to use what?
Use Deployment for stateless apps (APIs, web servers) — pods are interchangeable, any can be replaced. Use StatefulSet when each pod needs a stable identity, ordered startup/shutdown, or dedicated persistent storage (databases, message brokers, distributed systems).
